# MANUAL ANALYSIS PROTOCOL

## Purpose
Replace automated/template analysis with real, human-quality analysis by Astra.

## When Triggered
- Any link/PDF arrives in Telegram
- User explicitly requests analysis
- Content flagged as high-priority

## Protocol Steps

### 1. DEEP READ (5-10 minutes)
- Actually consume the content
- Read, don't skim (for articles under 5k words)
- For longer content: Strategic reading (intro, headers, conclusion, then depth where relevant)
- Take mental notes on: core thesis, key arguments, evidence quality, practical applications

### 2. INSIGHT EXTRACTION
Extract ONLY what you genuinely found:
- Core thesis (1 sentence)
- Key insights (3-5 bullets, specific not generic)
- Practical applications for Dibs (specific to his situation)
- Quality assessment (is this bullshit or legit?)
- Relevance score (1-10 for Dibs's goals)

### 3. HONEST ANALYSIS WRITEUP
Write in Astra voice (direct, no fluff):
- What this actually is
- What's valuable (specific points)
- What's questionable/critique
- Exact next steps Dibs should take
- Time investment required
- Opportunity cost

### 4. UI/UX HANDOFF
Pass to UI/UX Designer with:
- Full analysis text
- Relevance score
- Priority level
- Context for design choices
- Any special formatting needs

### 5. QUALITY CHECK (Astra)
Before Vercel deployment:
- [ ] Analysis is specific, not generic
- [ ] Contains at least 3 concrete insights
- [ ] Has actionable next steps
- [ ] Includes honest critique
- [ ] Relevance score included
- [ ] TL;DR is actually useful

### 6. DELIVERY TO DIBS
Send:
- Report URL
- TL;DR (2-3 sentences max)
- Relevance score: X/10
- One-line verdict
- Suggested action (read now / save for later / skip)

## Anti-Patterns (NEVER DO)
- ❌ "Analysis pending manual review"
- ❌ Generic bullet points
- ❌ Copy-paste from content
- ❌ No critique/only praise
- ❌ Fake insights
- ❌ Template language

## Success Criteria
Dibs reads the TL;DR and immediately knows:
1. What this content is about
2. Whether it's worth his time
3. What he'll get from it
4. One concrete action to take
